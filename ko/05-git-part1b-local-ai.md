# 05 - Git Part 1b: AI와 함께하는 로컬 Git

**이 섹션에서는 동일한 로컬 git 작업을 다시 해봅니다 — 하지만 이번에는 명령어를 직접 입력하는 대신 AI와 대화합니다.** 각 git 명령어가 무엇을 하는지 이해했으니, AI가 문법을 처리하도록 하고 여러분은 하고 싶은 것에 집중하면 됩니다.

---

## 1. 소개

이전 섹션에서는 모든 git 명령어를 직접 타이핑했습니다. 이것은 중요한 과정이었습니다 — 이제 스테이징, 커밋, diff가 실제로 무엇을 의미하는지 이해하게 되었으니까요.

하지만 핵심은 이것입니다: 그 명령어들을 외울 필요가 없습니다. 원하는 것을 자연어로 설명하면 LLM CLI 도구가 알맞은 명령어를 알아서 실행해 줍니다.

중요한 통찰은 이것입니다: **문법을 외우는 것보다 개념을 이해하는 것이 더 중요합니다.** "스테이징"이 무엇인지 알면 AI에게 "내 변경 사항을 스테이징해줘"라고 말할 수 있고, AI가 `git add`를 실행합니다. "커밋"이 무엇인지 알면 "스냅샷을 저장해줘"라고 말할 수 있고, AI가 무엇을 해야 하는지 압니다.

---

## 2. LLM CLI 도구 시작하기

iTerm2를 열고 `my-todo-list` 폴더(Part 1a에서 만든 폴더)로 이동한 다음 도구를 실행합니다:

```bash
cd my-todo-list
```

설치한 도구를 시작합니다:

| 도구 | 명령어 |
|------|--------|
| Claude Code | `claude` |
| Codex | `codex` |
| Gemini CLI | `gemini` |

**어떤 일이 생기나요:** 터미널에서 AI와 대화가 시작됩니다. 이제 명령어 대신 자연어를 입력할 수 있습니다.

> **팁:** 도구를 시작하기 전에 `my-todo-list` 폴더 안에 있는지 확인하세요. AI는 현재 폴더의 파일을 기준으로 작동합니다.

> **참고:**
> - [Claude Code](https://docs.anthropic.com/en/docs/claude-code) — Anthropic의 에이전틱 코딩 도구
> - [Codex CLI](https://developers.openai.com/codex/cli/) — OpenAI의 터미널 기반 코딩 에이전트
> - [Gemini CLI](https://github.com/google-gemini/gemini-cli) — Google의 오픈소스 터미널 AI 에이전트

## 3. 자연어 프롬프트와 Git 명령어 대응표

자연어가 배운 명령어와 어떻게 대응되는지 살펴봅시다:

| 하고 싶은 것 | AI에게 말하는 내용 | 실행되는 git 명령어 |
|---|---|---|
| 이 폴더 추적 시작 | "여기에 git 저장소를 초기화해줘" | `git init` |
| 사용자 정보 설정 | "git 이름을 홍길동, 이메일을 hong@example.com으로 설정해줘" | `git config user.name` + `git config user.email` |
| 변경 사항 확인 | "어떤 파일이 변경되었는지 보여줘" | `git status`, `git diff` |
| 파일 스테이징 | "TODO.md를 커밋 준비해줘" | `git add TODO.md` |
| 작업 저장 | "변경 사항을 'update to-do list' 메시지로 커밋해줘" | `git add` + `git commit` |
| 이력 보기 | "커밋 이력을 보여줘" | `git log` |
| 변경 사항 되돌리기 | "TODO.md의 변경 사항을 되돌려줘" | `git restore TODO.md` |
| 간결한 이력 보기 | "커밋 로그를 한 줄씩 간결하게 보여줘" | `git log --oneline` |

> **참고:** 정확히 이 문구를 사용할 필요는 없습니다. AI는 자연어를 이해하므로 "뭐가 바뀌었어?", "수정된 파일 있어?", "상태 보여줘" 등 다 통합니다.

## 4. 가이드 실습

Part 1a의 할 일 목록 시나리오를 다시 해봅시다. 이번에는 LLM CLI 도구를 사용합니다.

### 1단계: 현재 상태 확인

다음과 같이 입력합니다:

> "현재 git 상태와 커밋 이력을 보여줘"

AI가 `git status`와 `git log`를 실행하고 결과를 보여줍니다.

### 2단계: 새 파일 만들기

VSCode에서 `GOALS.md`라는 새 파일을 다음 내용으로 만드세요:

```markdown
# My Goals

## This Week
- [ ] Learn git basics
- [ ] Practice using the terminal
- [ ] Try an LLM CLI tool
```

`Cmd+S`로 저장합니다.

### 3단계: AI에게 변경 사항 물어보기

> "어떤 파일이 변경되었거나 추가되었어?"

AI가 `GOALS.md`가 새로운 추적되지 않은 파일이라고 보여줍니다.

### 4단계: 스테이징하고 커밋하기

> "새 GOALS.md 파일을 'Add weekly goals'라는 메시지로 커밋해줘"

AI가 파일을 스테이징하고 커밋을 만들어 줍니다.

### 5단계: 수정하기

VSCode에서 `GOALS.md`를 편집합니다 — "Learn git basics"를 완료 표시하고 새 목표를 추가합니다:

```markdown
# My Goals

## This Week
- [x] Learn git basics
- [ ] Practice using the terminal
- [ ] Try an LLM CLI tool
- [ ] Build something fun
```

`Cmd+S`로 저장합니다.

### 6단계: 변경 사항 확인하고 커밋하기

> "내가 뭘 바꿨는지 보여주고, 'Check off git basics, add new goal'이라는 메시지로 커밋해줘"

AI가 diff를 보여준 다음 커밋합니다.

### 7단계: 타임라인 보기

> "모든 커밋을 보여줘"

전체 이력이 표시됩니다 — Part 1a에서 직접 만든 커밋과 AI가 방금 만든 커밋 모두 볼 수 있습니다.

### 8단계: 되돌리기 연습

`GOALS.md`의 모든 내용을 삭제합니다 (전체 선택, 삭제, 저장). 그런 다음:

> "실수했어 — GOALS.md를 마지막 커밋 버전으로 복원해줘"

AI가 파일을 복구해 줍니다.

## 5. 핵심 통찰

지금 일어난 일을 보세요: Part 1a와 정확히 같은 git 작업 — init, status, add, commit, log, diff, restore — 을 했지만, 명령어를 기억할 필요가 없었습니다.

**개념을 이해하는 것이 중요합니다:**

- **스테이징** = 다음 스냅샷에 넣을 것을 고르기
- **커밋** = 스냅샷 찍기
- **Diff** = 무엇이 바뀌었는지 보기
- **복원** = 커밋하지 않은 변경 사항 되돌리기

이 개념들을 이해하면, 원하는 것을 자연어로 표현할 수 있습니다. AI가 여러분의 의도를 올바른 git 명령어로 변환합니다.

이것은 git을 넘어서 널리 적용되는 패턴입니다: **개념을 배우고, 문법은 AI에게 맡기세요.**

---

## 6. 명령어 직접 입력 vs. AI 사용

두 가지 접근 방식 모두 유효합니다. 간단한 가이드입니다:

| 상황 | 권장 방식 |
|------|-----------|
| 간단한 작업 (`git status`, `git log`) | 명령어 직접 입력 — 더 빠릅니다 |
| 기억나지 않는 복잡한 작업 | AI에게 요청 |
| 새로운 것을 배울 때 | AI에게 무엇을 하는지 설명해 달라고 요청 |
| 반복적인 작업 | AI에게 전체 과정을 요청 |
| 다음에 뭘 해야 할지 모를 때 | AI에게 안내를 요청 |

> **팁:** 명령어를 직접 배우고 싶다면 AI에게 "이걸 위한 git 명령어가 뭐야?"라고 물어볼 수 있습니다. 실행하지 않고 문법만 알려줍니다.
