# 03 - VSCode 안내

**이 섹션에서는 워크숍 전반에서 사용할 텍스트 편집기인 Visual Studio Code에 익숙해집니다.** 프로젝트의 모든 파일을 한꺼번에 보고 편집할 수 있는 "다중 파일 메모장"이라고 생각하세요. 약 5-10분이면 충분합니다.

---

## 1. VSCode 열기

VSCode를 여는 두 가지 방법이 있습니다:

**Spotlight에서 열기:**

1. `Cmd+Space`를 눌러 Spotlight 열기
2. "Visual Studio Code" 입력
3. Enter 누르기

**터미널에서 열기 (권장):**

작업할 폴더로 이동한 후 다음을 입력합니다:

```bash
code .
```

**어떤 일이 생기나요:** VSCode가 열리면서 사이드바에 해당 폴더가 표시됩니다. `.`은 "이 폴더"를 의미합니다 — CLI 섹션에서 이미 본 적이 있을 겁니다.

**확인 방법:** VSCode 창이 나타나고 사이드바 상단에 폴더 이름이 표시됩니다.

> **문제 해결:** "command not found: code"가 나타나면, VSCode를 수동으로 열고 `Cmd+Shift+P`를 누른 후 "shell command"를 입력하고 **"Shell Command: Install 'code' command in PATH"**를 선택하세요. iTerm2를 닫고 다시 연 다음 다시 시도해 보세요.

> **참고:** [VS Code Command Line Interface (CLI)](https://code.visualstudio.com/docs/configure/command-line)

---

## 2. 사이드바 (파일 탐색기)

VSCode 왼쪽에 프로젝트의 모든 파일과 폴더를 보여주는 열이 있습니다. 이것이 **파일 탐색기**입니다.

- 맨 왼쪽 아이콘 바의 맨 위 아이콘(겹쳐진 문서 모양)을 클릭하면 열 수 있습니다
- 폴더 옆의 화살표를 클릭하면 폴더를 펼칠 수 있습니다
- 이 사이드바는 컴퓨터의 실제 파일을 반영합니다 — 터미널에서 파일을 만들면 여기에도 나타납니다

> **팁:** 사이드바가 보이지 않으면 `Cmd+B`를 눌러 토글할 수 있습니다.

> **참고:** [VS Code User Interface](https://code.visualstudio.com/docs/getstarted/userinterface)

---

## 3. 파일 열기 및 편집

**파일 열기:** 사이드바에서 파일 이름을 클릭합니다. 메인 편집 영역에 파일이 열립니다.

**편집하기:** 파일 아무 곳이나 클릭하고 타이핑을 시작합니다. 일반 텍스트 편집기와 동일합니다.

**저장하기:** `Cmd+S`를 누릅니다.

**확인 방법:** 상단의 파일 이름 탭에 저장하지 않은 변경 사항이 있으면 점(또는 원)이 표시됩니다. `Cmd+S`를 누르면 점이 사라집니다.

> **팁:** 상단에 여러 탭이 있다면, 각 탭은 다른 파일입니다. 탭을 클릭하면 파일 간에 전환할 수 있습니다.

> **참고:** [VS Code Keyboard Shortcuts for macOS (PDF)](https://code.visualstudio.com/shortcuts/keyboard-shortcuts-macos.pdf)

---

## 4. 새 파일 만들기

**사이드바에서 만들기:**

1. 사이드바에서 폴더 이름 위에 마우스를 올리기
2. "새 파일" 아이콘(문서에 `+` 표시가 있는 모양) 클릭
3. 파일 이름을 입력하고 Enter 누르기

**메뉴에서 만들기:**

1. **File → New File** (또는 `Cmd+N`) 선택
2. 내용 입력
3. `Cmd+S`를 눌러 저장 — 이름과 위치를 선택하라는 창이 나타남

**확인 방법:** 사이드바에 새 파일이 나타나고 탭에 열립니다.

---

## 5. 통합 터미널

VSCode에는 내장 터미널이 있습니다. **View → Terminal** (또는 `` Ctrl+` ``)로 열 수 있습니다.

이 워크숍에서는 많이 사용하지 않을 예정입니다 — 메인 터미널로는 계속 iTerm2를 사용합니다. 하지만 온라인에서 언급되는 걸 볼 수 있으니 알아두면 좋습니다.

---

## 6. 마크다운 미리보기

워크숍에서 마크다운 파일(`.md`로 끝나는 파일)을 많이 작성하게 되므로, 유용한 기능을 하나 소개합니다:

**마크다운 파일 미리보기:**

1. VSCode에서 `.md` 파일 열기
2. `Cmd+Shift+V` 누르기

**어떤 일이 생기나요:** 미리보기 창이 열리면서 마크다운이 제목, 굵은 글씨, 체크박스 등으로 렌더링됩니다 — `#`이나 `*` 같은 기호 대신 깔끔하게 표시됩니다.

**확인 방법:** 새 탭에서 마크다운 파일이 보기 좋게 정리된 형태로 보입니다.

> **팁:** 원본 텍스트와 미리보기를 나란히 보려면 `Cmd+K`를 누른 다음 `V`를 누르세요 (`Cmd+K`를 먼저 누르고 손을 뗀 후 `V`를 누릅니다).

> **참고:** [Markdown and Visual Studio Code](https://code.visualstudio.com/docs/languages/markdown)

---

VSCode에 대해 지금 알아야 할 것은 이것이 전부입니다. 앞으로 iTerm2에서 git 명령어를 실행하면서 VSCode를 파일 편집기로 사용할 것입니다.
